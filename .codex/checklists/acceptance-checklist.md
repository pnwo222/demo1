# 通用验收检查表

## 总控流程验收

- [ ] Orchestrator 已声明当前阶段、调用 Agent、输入、输出、验收标准和下一阶段。
- [ ] 当前阶段产物已由 Orchestrator 汇总。
- [ ] 阶段门禁已检查。
- [ ] 需要用户确认的阶段已获得确认。

## 产品验收

- [ ] 当前需求文档路径已明确。
- [ ] HTML 版 PRD 已生成并可打开。
- [ ] 可交互低保真 HTML 原型已生成并可打开。
- [ ] 用户流程符合 PRD。
- [ ] 验收标准逐条通过。
- [ ] MVP 范围没有遗漏。
- [ ] 非本期范围没有被误做。

## UI 验收

- [ ] 已使用项目设计系统或 `.codex/skills/ui-ux-pro-max` 生成设计规则。
- [ ] 已连接 Figma 并生成可编辑设计图。
- [ ] Figma 设计图包含核心页面和关键状态。
- [ ] 正式 UI 是可落地设计稿，不是线框图或低保真原型。
- [ ] 目标设备或目标视口适配合理。
- [ ] 导航、弹窗、滚动区域和关键操作状态表现正常。
- [ ] Loading / Empty / Error / Offline 状态完整。
- [ ] 图片、icon、字体和 Handoff 标注满足设计交付要求。

## 功能验收

- [ ] 用户可以完成需求文档定义的主流程。
- [ ] 关键业务规则已按需求文档验证。
- [ ] 关键状态机或状态流转已验证。
- [ ] 异常流程、取消流程或失败流程已验证。
- [ ] 前端展示和后端最终校验职责清晰。

## 数据和质量验收

- [ ] Data Agent 已审查数据模型、索引、migration 和回滚策略。
- [ ] 核心实体、关系、快照或审计信息完整。
- [ ] 关键唯一约束和幂等约束明确。
- [ ] Unit / Integration / E2E / Contract Test 按范围通过。
- [ ] Lint / Typecheck / Build / Coverage 按范围通过。
- [ ] SAST / SCA / Secret Scan / DAST 按范围通过或有风险接受记录。

## 权限和安全验收

- [ ] 各角色只能访问授权范围内的数据。
- [ ] 敏感操作由后端或可信服务校验。
- [ ] 高风险接口有鉴权、签名、幂等或防重放策略。
- [ ] 日志不泄漏敏感信息。

## 发布验收

- [ ] 预发验证通过。
- [ ] 灰度观察正常。
- [ ] 回滚方案可用。
- [ ] 发布说明完整。
- [ ] 发布后监控正常。
