# 需求到原型页面蓝图模板

每个独立页面必须填写一份。字段必须来自需求、标书、PRD 或最小需求说明；需求未明确时写 `待确认`。

| 项 | 内容 |
| --- | --- |
| 需求编号 |  |
| 页面名称 |  |
| 菜单路径 |  |
| 路由路径 |  |
| 权限标识 |  |
| 可见角色 |  |
| 页面类型 | 标准 CRUD / 树形 CRUD / 内容管理 / 审核流程 / 只读查询 / 配置管理 / 授权分配 / 导入导出 |
| 参考 Snowy 页面 |  |
| 原始需求摘录 | 粘贴或摘要引用原始需求句子，不得只引用二次拆解表 |
| 原子需求清单 | 逐条列出字段、筛选、列表、状态、权限、操作和异常 |
| 查询字段 | 字段名、控件、默认值、是否必填 |
| 表格字段 | 字段名、展示形态、示例数据、排序/筛选 |
| 详情字段 | 字段名、展示形态、只读规则 |
| 新增字段 | 字段名、控件、必填、校验、默认值 |
| 编辑字段 | 字段名、控件、必填、校验、禁用规则 |
| 导入/导出字段 | 模板字段、导出字段、审批规则 |
| 操作按钮 | 查询、重置、新增、编辑、详情、删除、导入、导出、审核、授权、启停、更多 |
| 状态/异常 | 正常、空态、加载、失败、校验失败、无权限、审核中、已驳回等 |
| 权限差异 | 哪些角色可见/可点/不可见/禁用 |
| 字段展示形态 | 图片、头像、附件、金额、日期、状态标签、启停开关、脱敏、树形路径等 |
| 点击交互 | 每个按钮点击后的页面变化、弹窗、数据变化或反馈 |
| 覆盖矩阵行 | 对应覆盖矩阵编号 |

禁止将多个页面写成同一份通用蓝图。若页面很多，可以拆成多个蓝图文件或在 PRD/原型 HTML 中按页面分节列出。

字段完整性要求：

- 原始需求明确列出的筛选字段必须逐一进入“查询字段”。
- 原始需求明确列出的展示字段必须逐一进入“表格字段”和“详情字段”。
- 不得把多个状态合并成“状态”或“卡状态”；应保留业务名，例如“专区注册状态、社保绑卡状态、省内持卡状态、卡规范版本、社保卡金融账户激活状态”。
- 如果某个字段因为页面空间未展示在主表格，必须进入详情抽屉或列设置，并在覆盖矩阵说明。
