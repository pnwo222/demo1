## 审查修复（第二轮）

- 节点标注改用稳定 `nodeKey`：由页面 ID 与查询字段、操作名称、业务 `rowKey`、字段名共同组成，不再依赖运行时随机属性。持久化状态增加当前页面、当前角色和 `nodeKey`；刷新、页面切换及“另存为”重开均恢复相同 `commentId/nodeKey` 的 pin。
- 专属动作按语义拆分：所有包含“导入”的动作进入文件选择、预览/移除、预校验和逐项错误反馈；授权展示当前范围、组织/资源树、变更范围和越权拦截；审核要求通过/驳回与审核意见并更新行状态；启停/状态切换二次确认后更新行状态并提示审计留痕；远程配置、导出、上传继续使用独立表单或弹窗。
- 增加当前角色预览切换和逐页按钮权限过滤。PAM-002 中安全审计人员仅保留只读详情，PAM 分级子账户才显示发起采集与更新登记状态；审核人员、内容运营人员、设备运维人员按职责过滤操作。
- 每页查询枚举由页面模型显式提供，例如审核状态、认证结果、通行方向、登记状态；日期范围使用日期对象按日边界比较；每页生成 28 条唯一 `rowKey` 数据并按实际筛选总数分页。
- HTML 标题调整为“宁波市高校一卡通专区后管低保真原型”。自动标注删除按不可变基线处理：未修改基线只在本地隐藏，修改后的自动标注删除时恢复基线，用户标注直接删除，并提供恢复隐藏自动标注入口。

## 复审修复（第三轮）

- 权限由页面模型中的 `actionPolicy` 逐页、逐角色声明，运行时不再使用角色名关键词粗略排除。生成审计覆盖 49 页全部“角色 × 动作”组合，并校验策略动作均来自蓝图。公告页内容运营人员可见新增、编辑、删除、提交审核、查看详情；审核人员仅可见审核、查看详情。
- “提交审核”与“审核”拆成两个独立状态机：内容运营人员确认发布范围并填写提交说明后，草稿变为待审核；审核人员填写审核意见并选择通过或驳回后，状态变为审核通过或审核驳回。“提交审核”不会进入审核人员弹窗。
- 启停状态值按页面语义映射：平台用户管理使用启用/禁用；应用可用状态等页面使用启用/停用。操作弹窗展示当前值和目标值，确认后写入对应页面枚举值。
- 补全并以下拉呈现蓝图枚举查询字段，包括可见身份、展示模式、身份标签、民族、性别、认证场景等；生成审计确保全部蓝图枚举/状态查询字段都有非空 `options`。

## 权限数据污染修复（第六轮）

- 根因：PAM-008 是最后一个页面，Markdown 表解析器只在三级/四级标题处停止，没有在后续 `## 覆盖矩阵` 二级标题处停止，导致覆盖矩阵表头和 49 个需求编号被误收为 `actionPermissions`。
- 修复：表解析统一在二至四级标题处终止；PAM-008 现在仅保留查询、重置、查看日志详情、配置分级权限四个真实权限键。
- 增加全量门禁：每页 `actionPermissions` 只能包含该页业务动作及查询/重置，所有业务动作必须存在权限行；独立 Node 审计对最终蓝图与 HTML 做精确比对。
