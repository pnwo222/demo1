需求来源：docs/requirements/food-delivery-platform.md

点击路径：附近商家 -> 进店 -> 加购 -> 购物车 -> 去结算 -> 提交订单 -> 模拟支付 -> 商家/骑手状态流转。
  • 金额展示为前端预估，提交订单后模拟后端重算。
  • 支付成功和重复回调演示幂等规则。
  • 订单状态按钮演示明确状态机，不允许非法跳转。
  • 切换不可配送地址后，提交订单会被阻断。
  • 越权访问演示后端权限校验必须存在。
