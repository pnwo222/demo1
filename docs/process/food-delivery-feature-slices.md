## 2. 拆分原则

- 按用户价值和可验收业务闭环切分，不按岗位粗暴拆分。
- 每个 slice 同时包含前端、后端、数据和测试要求。
- 金额、支付、退款、库存、权限、状态机相关 slice 默认按 P0/P1 高风险处理。
- API 契约和数据模型稳定前，前后端可基于 mock 或 contract 并行，但不能绕过后端可信校验。
- QA 和 Security 从早期 slice 开始准备测试和审查，不等开发完成后再介入。

## 4. Slice 明细

### FS-01 商家浏览与配送范围

- 用户价值：用户打开 H5 后能基于地址看到可配送商家、营业状态、起送价和配送费。
- 前端任务：
  - 实现附近商家页、地址栏、商家卡片、休息中和不可配送禁用状态。
  - 覆盖 loading、empty、error 状态。
- 后端任务：
  - 提供商家列表和详情 API。
  - 根据地址、营业状态和配送规则返回可配送状态。
- 数据库影响：
  - 使用 `fd_merchant`、`fd_merchant_business_time`、`fd_delivery_rule`、`fd_user_address`。
  - 需要商家状态和营业状态索引。
- 测试要求：
  - 商家开放、休息、禁用、地址不可配送。
  - 用户只能使用自己的地址。
- 验收标准：
  - 商家列表展示营业状态、配送费、起送价和是否可下单。
  - 不可配送或休息商家不能进入下单路径。
- 依赖关系：无业务前置，依赖基础数据 seed。
- 风险和回滚：
  - P1。可通过隐藏 H5 入口或回滚商家列表接口灰度。

### FS-02 商品 SKU 与购物车

- 用户价值：用户可进入商家，选择可售 SKU 加入购物车并调整数量。
- 前端任务：
  - 实现商家详情、商品分类、商品卡片、库存不足和下架状态。
  - 实现购物车加购、减购、清空和金额预估。
- 后端任务：
  - 提供商品/SKU 查询 API 和购物车 CRUD API。
  - 加购时校验 SKU 上架、商家一致、库存提示。
- 数据库影响：
  - 使用 `fd_product_category`、`fd_product`、`fd_sku`、`fd_inventory`、`fd_cart_item`。
  - `uk_cart_user_sku(user_id, sku_id)` 防重复行。
- 测试要求：
  - SKU 下架、库存不足、跨商家购物车、重复加购。
  - 购物车用户归属越权。
- 验收标准：
  - 可售 SKU 可加购；下架和库存不足状态清晰。
  - 购物车数量和预估金额可更新。
- 依赖关系：依赖 FS-01。
- 风险和回滚：
  - P1。回滚时保留购物车表，禁用加购入口。

### FS-03 订单预览与后端重算金额

- 用户价值：用户提交前能看到后端可信金额、优惠、配送费和包装费明细。
- 前端任务：
  - 实现确认订单页、地址展示、费用明细和金额变化提示。
  - 前端金额只展示预估，使用后端 preview 结果覆盖。
- 后端任务：
  - 提供 `POST /api/h5/orders/preview`。
  - 校验商家营业、SKU 上架、库存、地址可配送、优惠可用。
  - 返回费用明细和预览版本。
- 数据库影响：
  - 读取购物车、SKU、库存、优惠券和配送规则。
  - 暂不创建订单，必要时可记录短期 preview token 或版本。
- 测试要求：
  - 前后端金额不一致、优惠失效、地址不可配送、库存不足。
- 验收标准：
  - 确认页展示商品小计、优惠、配送费、包装费、应付金额。
  - 金额以后端 preview 为准。
- 依赖关系：依赖 FS-02。
- 风险和回滚：
  - P0。金额错误阻断上线；可回滚确认订单入口。

### FS-04 创建订单、库存锁定和费用快照

- 用户价值：用户可基于确认页创建待支付订单，并锁定库存防止超卖。
- 前端任务：
  - 提交订单时携带 preview 版本和幂等键。
  - 处理创建失败原因，引导回购物车或刷新金额。
- 后端任务：
  - 提供 `POST /api/h5/orders`。
  - 事务内创建订单、明细、费用快照、状态日志，锁定库存。
  - 使用 `uk_order_user_idem(user_id, idempotency_key)` 防重复下单。
- 数据库影响：
  - 写入 `fd_order`、`fd_order_item`、`fd_order_fee`、`fd_order_status_log`、`fd_inventory_log`。
  - 更新 `fd_inventory.available_qty` 和 `locked_qty`。
- 测试要求：
  - 重复提交、并发库存、库存释放、金额快照。
  - 订单创建失败不产生半成品数据。
- 验收标准：
  - 成功创建待支付订单，费用来源可追溯。
  - 重复提交只产生一个有效订单。
- 依赖关系：依赖 FS-03。
- 风险和回滚：
  - P0。需要事务和补偿；回滚时保留表，禁用下单入口并释放过期锁定库存。

### FS-05 支付单与支付回调幂等

- 用户价值：用户支付后订单可靠进入已支付，不会重复扣款或重复状态流转。
- 前端任务：
  - 发起支付、展示待支付和支付结果。
  - 轮询或接收支付结果后跳转订单详情。
- 后端任务：
  - 创建支付单。
  - 支付回调验签、校验金额、订单号、渠道流水。
  - 幂等更新支付单和订单状态，确认库存扣减。
- 数据库影响：
  - 写入 `fd_payment_order`、`fd_payment_callback_log`、`fd_order_status_log`、`fd_audit_log`。
  - 更新 `fd_order.pay_status` 和 `status`。
- 测试要求：
  - 重复回调、伪造签名、金额不一致、订单号不一致、已取消订单回调。
- 验收标准：
  - 合法回调使订单从待支付到已支付。
  - 重复回调返回成功但无重复副作用。
- 依赖关系：依赖 FS-04。
- 风险和回滚：
  - P0。支付入口必须灰度；回滚不得删除回调日志。

### FS-06 商家履约状态流转

- 用户价值：商家能接单、标记制作、出餐，用户看到履约进度。
- 前端任务：
  - 商家订单列表和操作入口。
  - H5 订单详情展示状态变化。
- 后端任务：
  - 提供商家订单列表、接单、制作中、待配送 API。
  - 校验商家门店归属和状态机合法性。
- 数据库影响：
  - 更新 `fd_order.status`，写 `fd_order_status_log` 和 `fd_audit_log`。
- 测试要求：
  - 商家越权、重复接单、非法状态跳转。
- 验收标准：
  - 商家只能处理自己的订单。
  - 状态只能按已支付 -> 商家已接单 -> 制作中 -> 待配送流转。
- 依赖关系：依赖 FS-05。
- 风险和回滚：
  - P1。可禁用商家操作入口，保留订单状态。

### FS-07 配送任务与完成

- 用户价值：骑手可处理配送任务，用户可查看配送中和已完成。
- 前端任务：
  - 骑手任务列表和状态操作入口。
  - 用户订单详情展示待配送、配送中、已完成。
- 后端任务：
  - 创建或分配配送任务。
  - 骑手取餐、配送中、送达完成 API。
  - 校验骑手任务归属和状态机。
- 数据库影响：
  - 写入 `fd_delivery_task`，更新 `fd_order.status`，写状态日志。
- 测试要求：
  - 骑手越权、重复送达、未分配任务操作。
- 验收标准：
  - 骑手只能操作自己的任务。
  - 配送中到已完成后用户可进入评价入口。
- 依赖关系：依赖 FS-06。
- 风险和回滚：
  - P1。配送入口可灰度；状态回滚需人工异常单处理。

### FS-08 取消、退款和退款回调

- 用户价值：用户在允许状态可取消或申请退款，资金状态可追踪。
- 前端任务：
  - 取消订单、申请退款、退款中和已退款状态展示。
  - 异常原因提示。
- 后端任务：
  - 取消订单 API，释放锁定库存。
  - 退款申请 API，创建退款单。
  - 退款回调验签、金额校验和幂等处理。
- 数据库影响：
  - 写入 `fd_refund_order`、`fd_refund_callback_log`、`fd_order_fee` 退款明细、审计日志。
  - 更新订单和库存状态。
- 测试要求：
  - 重复退款、退款金额错误、退款回调重复、已完成订单退款限制。
- 验收标准：
  - 取消和退款只在允许状态发生。
  - 重复退款回调无重复资金副作用。
- 依赖关系：依赖 FS-04、FS-05。
- 风险和回滚：
  - P0。退款功能必须灰度；回滚不删除退款单和回调日志。

### FS-09 审计、异常订单和运营处理

- 用户价值：平台能追踪关键操作并处理异常订单，降低资金和状态风险。
- 前端任务：
  - 运营异常订单列表、详情和处理结果展示。
  - 审计日志查询入口。
- 后端任务：
  - 记录支付、退款、取消、状态流转、越权拦截审计。
  - 提供异常订单查询和处理 API。
- 数据库影响：
  - 写入 `fd_audit_log`、`fd_abnormal_order`。
- 测试要求：
  - 审计完整性、越权拦截记录、异常单状态流转。
- 验收标准：
  - P0/P1 操作均有可追踪审计记录。
  - 运营可按状态处理异常单。
- 依赖关系：依赖 FS-04 至 FS-08。
- 风险和回滚：
  - P1。审计日志不可回滚删除；运营入口可禁用。

### FS-10 质量门禁和安全回归

- 用户价值：主链路上线前通过核心质量、安全和回归验证。
- 前端任务：
  - 移动端 H5 主流程 smoke，覆盖 390x844、375px 等视口。
  - loading、empty、error、disabled 状态回归。
- 后端任务：
  - 单元、集成、合同测试。
  - 支付回调、订单状态机、库存并发、权限越权测试。
- 数据库影响：
  - migration 验证、回滚演练和数据一致性检查脚本。
- 测试要求：
  - Lint、Typecheck、Unit、Integration、E2E、API Contract、Build、Coverage。
  - SAST、SCA、Secret Scan、DAST。
- 验收标准：
  - P0/P1 问题清零。
  - CI 和安全门禁通过或有明确风险接受记录。
- 依赖关系：依赖全部业务 slice。
- 风险和回滚：
  - P0。未通过不得合并或全量发布。

## 6. 推荐并行 Wave

| Wave | 目标 | 可并行工作 |
| --- | --- | --- |
| Wave 0 | 补齐契约和测试设计 | API contract、状态机规则、QA 测试计划、安全测试计划 |
| Wave 1 | 基础浏览和商品购物车 | FS-01、FS-02 的前后端和数据 seed |
| Wave 2 | 下单和金额库存 | FS-03、FS-04，QA 并行准备金额、库存、幂等测试 |
| Wave 3 | 支付、履约、配送 | FS-05、FS-06、FS-07，Security 并行审查回调和权限 |
| Wave 4 | 退款、审计、质量门禁 | FS-08、FS-09、FS-10 |
