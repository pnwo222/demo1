# Snowy 原型 Demo 运行时组件化设计

## 目标

保留原始 Demo 的页面内容、视觉样式和交互能力，将最终入口改为通过本地文件引用样式、状态、动作及 Vue 预设组件。`index.html` 可直接双击打开，不依赖 npm、Vite、HTTP 服务、ES Module 或 `fetch`。

## 架构

- `index.html`：只包含挂载节点、CDN 依赖和本地脚本/样式引用。
- `styles/snowy-prototype.css`：原始 Demo 公共样式。
- `app/prototype-data.js`：系统信息、示例数据和页面需求基线。
- `app/prototype-state.js`：根应用状态、业务动作、上传和标注逻辑，向组件提供共享上下文。
- `components/*.js`：使用普通脚本注册全局 Vue 组件；组件通过 `inject` 使用根应用共享上下文。
- `app/main.js`：注册组件、创建根应用并挂载。

## 预设组件

现有 Snowy 壳、标注工具栏、查询区、数据表格、菜单资源页、组件预设页、节点评论输入、页面需求抽屉、内容表单抽屉、菜单表单抽屉、导入弹窗、列设置弹窗、标注编辑弹窗和图片预览均作为预设组件。

后续需求优先复用预设组件。没有满足需求的组件时，必须先读取当前 Snowy 框架真实页面和本 Demo，复制最接近组件的结构、样式及交互扩展新组件，并登记组件用途、来源和验证项。

## 兼容约束

- 使用经典 `<script src>`，不使用 `type="module"`。
- 本地脚本按数据、组件、状态、入口的依赖顺序加载。
- 原始 Demo 金标保持不变，用于内容、能力和视觉回归。
- 最终入口不得重新内嵌完整 CSS、根模板或业务运行时代码。

## 验收

- 双击 `index.html` 可正常加载。
- 入口引用外部 CSS、数据、状态、入口及全部预设组件。
- 页面仍包含原 Demo 的菜单、查询、表格、组件预设、抽屉、弹窗和标注能力。
- 校验器能够识别缺失组件引用、加载顺序错误和重新内嵌的大段实现。
