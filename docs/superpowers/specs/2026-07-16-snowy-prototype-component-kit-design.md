# Snowy 原型组件套件设计

## 目标

将当前单体 Demo 拆为可复用、可校验、可构建的 Snowy 原型组件套件。后续 Agent 只编写逐页业务 Schema，已有组件必须复用；缺失组件必须按 Snowy 设计令牌实现并注册。构建结果仍为可直接打开、可另存为、包含标注状态的单 HTML。

## 架构

组件套件由四层组成：Snowy 设计令牌与核心样式、Vue/Ant Design Vue 组件注册表、固定标注运行时、显式页面 Schema。`build-prototype.mjs` 读取这些源文件和业务 Schema，校验组件名称、字段声明和交互后，将全部内容内联到目标 HTML。

组件注册表包含：管理端外壳、查询表单、工具栏、数据表格、分页、详情抽屉、表单抽屉、确认弹窗、导入弹窗、图片上传、附件上传、文本/脱敏文本/金额/日期时间/状态标签/开关/图片/附件/进度/徽标等字段组件，以及标注工具栏、节点覆盖层、评论输入、标注详情、页面需求抽屉和另存为功能。

## 页面 Schema

每个独立页面分别声明菜单、路由、权限、页面类型、查询字段、表格列、详情字段、创建字段、编辑字段、工具栏动作、行操作、状态和标注。每个字段必须显式声明 `component`，不得根据字段名称正则猜测。查询、表格、详情、创建和编辑不可通过“同上”或自动复制隐式生成。

```js
{
  id: 'student',
  queryFields: [{ key: 'name', label: '姓名', component: 'Input' }],
  columns: [{ key: 'idCard', title: '身份证号', component: 'MaskedText' }],
  detailFields: [{ key: 'idCard', label: '身份证号', component: 'MaskedText' }],
  createFields: [],
  editFields: []
}
```

## 扩展组件

注册表不存在需求所需组件时，在 `src/extensions/` 新建组件并登记来源、用途和设计基类。扩展组件必须使用 Snowy CSS 变量和 Ant Design Vue 基础组件，不得覆盖核心类。未注册组件、重复组件名、原生文件输入、未知交互和缺失字段组件声明均阻塞构建。

## 单 HTML 构建

构建器不依赖前端工程安装，只使用 Node.js 标准库。它将模板、核心 CSS、组件运行时、标注运行时、组件清单、页面 Schema 和种子数据内联到一个 HTML。生成文件保留 CDN 依赖，但不依赖本地 JS/CSS 文件；“另存为”会继续嵌入最新标注和页面需求状态。

## 验证

静态校验检查组件清单版本、受保护源码片段、组件来源、Schema 字段组件声明、页面专属字段集合、禁止模式和需求覆盖。Node 测试覆盖组件注册、未知组件失败、缺失组件声明失败、构建单 HTML、核心组件内联和标注运行时内联。运行时检查继续验证页面打开、查询重置、表格、抽屉、上传和标注主链路。

## 性能

需求较多时，Agent 按模块生成独立页面 Schema，不直接编辑共享 HTML。构建器一次汇总，避免多个 Agent 重写同一文件，也避免为 49 个页面重复生成相同组件代码。
