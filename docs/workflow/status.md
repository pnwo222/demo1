## 状态值

阶段状态只能使用以下值：

| 状态 | 含义 |
| --- | --- |
| 未开始 | 尚未进入该阶段 |
| blocked_until_developer_confirmed_ready | 等待开发者确认前后端可运行 |
| 需确认 | 等待开发者决策或确认 |
| 已开始 | 阶段已进入，产物未完成 |
| 已跳过 | 开发者明确跳过该阶段 |
| 已完成 | 阶段产物已完成并记录 |
| 阻塞 | 阶段存在阻塞，需要处理 |

## 阶段记录规则

Orchestrator 在每个阶段进入、完成、跳过或阻塞时，必须更新本文件。

每次更新至少记录：

```text
阶段:
状态:
来源:
产物:
下一步:
时间:
```

## 框架运行自检

开发者确认前后端可运行后，Orchestrator 必须更新：

```text
框架运行自检: developer_confirmed_ready
前端运行确认: 已确认
后端运行确认: 已确认
确认来源: 开发者回复“前后端已确认可运行”或等价表达
确认时间: <YYYY-MM-DD HH:mm>
```

未确认前，状态必须保持为 `blocked_until_developer_confirmed_ready`，不得进入 PRD/UI/技术设计或开发阶段。

## 跳过记录

如开发者跳过 PRD、原型或 UI/Figma，Orchestrator 必须在这里追加记录：

```text
跳过项:
跳过原因:
最小需求说明:
验收标准:
风险记录:
记录时间:
```
