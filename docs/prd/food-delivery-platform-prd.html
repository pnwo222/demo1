外卖平台产品需求文档

当前需求文档路径：docs/requirements/food-delivery-platform.md

项目定位面向移动端用户体验的外卖平台，前端为 Vue 3 H5，后端为 Java 服务端。
核心链路浏览商家 -> 选择商品 -> 加入购物车 -> 下单 -> 支付 -> 商家接单 -> 配送 -> 完成/取消/退款。
阶段目标定义 MVP 产品范围、业务规则、验收标准和低保真原型输入。
高风险主题金额、支付幂等、库存一致性、订单状态机、权限隔离、审计日志。

1. 背景与目标

本期建设一个可支撑外卖主链路的产品基础能力，覆盖用户点餐、购物车、下单、支付、商家履约、配送状态和订单结束态。产品设计优先保障下单链路稳定、金额计算正确、库存与订单状态一致、角色权限清晰，以及移动端 H5 使用顺滑。

产品成功定义：用户可以在移动端完成从选店、选品、下单、支付到查看配送状态的闭环；商家、骑手和平台角色能在权限范围内处理各自任务；关键资金、库存和状态流转均有后端可信校验与审计记录。

2. 用户角色

角色目标关键操作权限边界
用户快速完成点餐并跟踪订单浏览商家、选 SKU、管理购物车、下单、支付、取消、申请退款、评价只能查看和操作自己的订单与地址

管理员配置系统并保障安全运维权限配置、系统配置、安全与运维处理具备最小必要权限和审计要求

3. 核心业务流程

  1. 用户进入 H5 首页，基于地址浏览可配送商家列表。
  2. 用户进入商家详情，按类目选择商品和 SKU，加入购物车。
  3. 用户在购物车确认商品、数量、优惠、配送费、包装费和预估金额。
  4. 用户提交订单，后端重新校验商家营业、商品上架、库存、配送地址和最终金额。
  5. 用户发起支付，支付回调由后端校验签名、金额、订单号和幂等性。
  6. 商家接单并更新制作状态，系统生成或分配配送任务。
  7. 骑手更新配送状态，用户查看进度。
  8. 订单完成后用户可评价；异常情况下走取消或退款流程。

金额和支付规则

规则产品要求验收点
后端可信金额前端只能展示和预估，提交订单后以后端重算为准确认页必须提示金额以提交结果为准
金额精度不能用浮点数直接计算金额，应使用整数分或 Java BigDecimal技术设计和测试覆盖精度用例
金额来源支付金额、优惠、配送费、包装费、退款金额必须有明细来源订单确认页和订单详情页展示费用明细
支付回调必须校验签名、金额、订单号和幂等性重复回调不能重复扣款或重复流转状态
审计日志支付、退款、取消订单必须记录审计日志后台可追踪关键操作

库存和购物车规则

场景规则异常处理
加购展示库存和上下架状态，允许购物车实时预估库存不足时禁用或提示调整数量
下单后端校验商品上架、库存、商家营业、地址可配送失败时返回可读原因，并引导用户返回购物车
支付确认库存扣减、订单创建、支付确认之间必须有一致性策略支付失败或超时需要释放锁定资源
重复操作防止重复下单、重复支付、重复退款幂等键或业务唯一约束保证只生效一次

7. 非功能需求

  • 移动端 H5 首屏信息结构清晰，核心点餐路径不超过三层页面。
  • 列表、详情、购物车、订单确认和订单详情需覆盖 loading、empty、error 状态。
  • 后端接口必须可测试、可监控、可回滚，关键操作具备审计日志。
  • 权限判断以后端为准，前端隐藏按钮不作为安全边界。
  • 核心链路必须具备幂等、事务或补偿设计，支持重复请求和并发场景。

8. 验收标准

ID验收项验收标准优先级
AC-01商家浏览用户可看到可配送商家、营业状态、配送费和起送信息P1
AC-02商品加购用户可按 SKU 加入购物车，库存不足或下架商品不可正常下单P1
AC-03订单确认确认页展示商品、地址、优惠、配送费、包装费和应付金额来源P0
AC-04后端金额校验创建订单时以后端金额为准，前端金额不一致时必须重新确认P0
AC-05支付幂等重复支付回调不会重复扣款、重复生成状态流转或破坏订单数据P0
AC-06订单状态机状态只能按允许路径流转，非法状态跳转被拒绝并记录P0
AC-07权限隔离用户、商家、骑手、运营只能访问权限范围内数据P0
AC-08取消退款取消和退款展示状态、金额和处理结果，关键操作有审计日志P1
AC-09移动端体验核心 H5 页面在主流移动端 viewport 下无遮挡、可点击区域合理P1
AC-10异常提示库存不足、商家休息、地址不可达、支付失败等异常有明确引导P1

10. 风险和不在本期范围

核心风险：金额错误、重复扣款、越权操作、订单状态错乱和库存并发超卖均按 P0/P1 重点审查，设计、开发、测试和发布阶段必须持续跟踪。

本期不做

  • 复杂会员成长体系、积分商城、预付卡和充值余额。
  • 复杂智能调度、路径优化和骑手端完整 App。
  • 跨城、多门店集团、供应链和采购管理。
  • 复杂广告推荐、直播、社区内容等非点餐主链路能力。
  • 真实第三方支付渠道接入细节，由技术设计阶段另行定义接口契约和回调安全策略。
