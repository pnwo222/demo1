背景与目标

建设高校一卡通学校专区及配套学校端、平台管理端、校园接口和 PAM 人员设备协同能力，使学校可运营专区内容、查询同步数据、监控设备，平台可治理组织权限和审计链路，接口可安全承载认证及交易数据，PAM 可管理人员、分组、掌静脉特征状态与设备运维。

目标包括：统一学校专区运营入口；打通校园数据中台、宁波一卡通、社保卡、门禁、图书和 PAM 数据；建立学校数据隔离、最小权限、敏感数据保护和全链路审计；对交易、批量、授权及远程配置实施高风险控制。

角色 可见范围 权限边界

学校管理员 本校师生、内容、记录、设备、用户 强制学校数据范围隔离，不得跨校查询

前端按钮隐藏不能替代后端鉴权。组织范围、角色、资源、按钮权限和敏感字段权限必须在服务端校验，越权访问返回明确无权限结果并写安全日志。

总体业务流程

  1. 校园数据中台或外部系统同步师生、记录和申领数据，学校端按本校范围查询详情。
  2. 内容运营人员维护 Banner、应用、首页模块、公告、文章和活动；需审核内容经审核后发布并同步专区。
  3. 平台管理员维护学校、用户、角色、模块和菜单；权限变更即时审计并按策略生效。
  4. 接口调用方经鉴权调用初始化、人员、认证和交易接口；平台完成校验、幂等、处理、重试及日志记录。
  5. PAM 管理人员与设备分组、特征状态、通行同步和运维；导入先校验，授权与远程配置二次确认，结果可追溯。

状态与业务规则

  • 通用页面覆盖加载中、正常、空数据、查询无结果、接口/同步异常和无权限；表单页面另覆盖校验失败、保存中、成功、失败。
  • 内容状态不合并：可用/停用、展示/隐藏、审核状态、推荐状态和发布状态分别表达；审核流至少包含草稿、待审核、通过、驳回、已发布/下线。
  • 同步记录保留来源系统、最近同步时间、结果和失败原因；只读数据不得在本地以编辑方式掩盖源数据问题。
  • 删除、批量操作、权限授权、敏感导出和远程配置必须二次确认，并显示影响范围与执行结果。
  • 设备覆盖在线、离线、故障、通信中断和业务报警；监控页面不因“查看异常”自动获得远程控制权。
  • 上传支持选择、预览、移除及类型/大小校验；导入支持预校验、重复项、异常项和结果反馈。

敏感数据与安全规则

  • 身份证号、手机号、社保卡号在列表脱敏；明文详情和导出需单独权限并记录访问人、时间、条件、用途和结果。
  • 掌静脉特征是生物识别敏感数据：页面只展示特征标识、状态、数量和时间，不展示真实模板；后续技术方案落实传输/存储加密、密钥管理、隔离、留存和销毁。
  • API 使用调用方身份、接口权限、时间戳/随机数、防重放、限流和审计；认证凭证及敏感载荷禁止落普通日志。
  • API-007、API-008 必须落实签名、幂等键、流水唯一、金额精度与范围校验、重复推送处理、失败重试、对账和人工处置入口。
  • 批量导入导出需最小数据范围、文件安全校验、失败明细和审计；权限变更需展示变更前后范围，防止操作者给自己越权提权。
  • 设备远程配置校验设备范围和参数，记录配置前后值、操作者、时间、目标、结果；失败不得伪装成功，批量执行支持部分成功明细。

异常场景

场景 预期处理
外部同步超时/中断 保留旧数据并标注新鲜度，记录失败原因，提供有权限的重试入口
重复或乱序交易 以业务流水/幂等键拒绝重复，记录冲突并进入对账
金额非法或签名失败 拒绝处理，不改变账务状态，写安全审计并告警
越权查询/操作 服务端拒绝，前端提示无权限，记录主体、资源和来源
导入包含重复/错误项 执行前预览统计；支持全部拒绝或按确认策略处理，输出逐项结果
导出超范围 拒绝或收窄到授权范围；不得依赖前端过滤
远程配置部分失败 显示逐设备结果，成功项留痕、失败项可安全重试
生物特征不可用 不暴露模板，显示受控状态并转人工采集/排障流程
页面无数据/无权限 空状态与无权限状态分开，不误报系统错误

非功能需求

  • 安全：遵循最小权限、学校数据隔离、敏感数据最小化、全链路审计；上线前完成越权、接口滥用、密钥与依赖检查。
  • 性能：列表分页；大数据导出采用受控异步任务；实时通行与设备监控指标在技术方案定义容量、延迟与降级策略。
  • 可靠性：外部调用设超时、重试、熔断和告警；交易与同步处理可幂等恢复；关键配置具备回滚记录。
  • 可运维：接口、同步、交易、设备和远程配置具备关联标识、状态、耗时、错误摘要和告警入口。
  • 可用性：查询条件可重置，状态语义一致，危险操作明确提示，表单错误定位到字段。
  • 兼容性：后管遵循现有 Snowy Vue 3 框架和支持的桌面浏览器；H5 兼容性延期确认。
